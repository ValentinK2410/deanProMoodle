# Запись в когорту из WordPress

Сайт на WordPress (плагин course-plugin) после регистрации пользователя вызывает Moodle Web Services:

- `core_user_create_users` / поиск по email — создание или сопоставление пользователя;
- `core_cohort_add_cohort_members` — добавление в глобальную группу (когорту), ID когорты задаётся в карточке программы в WP (`_program_moodle_cohort_id`);
- `core_message_send_instant_messages` — уведомление пользователю в Moodle (при включённых настройках уведомлений письмо может дублироваться на email).

**Настройка Moodle:** внешний сервис / токен должен включать перечисленные функции. Права роли пользователя токена — на управление когортами и отправку сообщений.

Плагин `local_deanpromoodle` не обязан дублировать эту логику: запись в когорту выполняется со стороны WordPress.
